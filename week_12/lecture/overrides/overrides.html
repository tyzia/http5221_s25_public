DevTools Sources Tab: Overrides Demo

Lecture: Understanding Overrides

Overrides allow you to:

  • Make persistent changes to files directly in DevTools
  • Test fixes without modifying original source files
  • Debug CSS, HTML, and JavaScript in real-time

Current State: CSS Not Loaded

This red dashed border indicates our broken-styles.css failed to load.

Check the Network tab to see the 404 error.

Success State: CSS Loaded!

This green border appears when you fix the CSS path using Overrides.

Hands-On Exercise: Using Overrides

Step 1: Enable Overrides

  1. Open DevTools (F12 or Ctrl+Shift+I)
  2. Go to Sources tab
  3. Select Overrides (click >> if hidden)
  4. Choose an empty folder on your computer
  5. Click Allow when prompted
Tip: This creates a local copy where DevTools saves your changes.

Step 2: Fix the CSS Path

  1. In Sources tab, find this HTML file under Page
  2. Change href="broken-styles.css" to href="styles.css"
  3. Save (Ctrl+S/Cmd+S)
  4. Refresh the page - changes will persist!
Note: The green success section should now appear.

Step 3: Verify in Network Tab

  1. Go to Network tab
  2. Refresh and confirm styles.css loads with status 200
  3. Notice the 404 error for broken-styles.css is gone

Troubleshooting

  • Overrides not working? Try:
    • Ensure you selected a folder and clicked "Allow"
    • Check Chrome is updated to latest version
    • Restart DevTools if changes don't appear
  • Want to reset? Disable overrides in Sources tab or delete the override folder.
